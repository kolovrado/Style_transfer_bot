1. В папке models запускаем download_models.py, который загружает нужные модели (оставил используемую по умолчанию VGG-19)
2. В файле config.py вписываем свой токен бота
3. Запускаем bot.py
4. Переходим в телеграм бота (например, t.me/k201320_bot )
5. Отправляем картинки. Первая - изменяемая, вторая - стиль. Быстрый способ - стандартный на предобученной модели из библиотеки tensorflow и подходит для цветовой стилизации. Медленный проводит полный цикл обработки и делает более приближенные стилизации (картины из макарон и т.д.)
6. У бота есть возможность образной связи. Все сообщения, содержащие слово "вопрос" сохраняются в txt файл QA.
7. Слово "пока" удаляет все данные пользователя. Для отладки есть возможность выключить бота, для этого необходимо раскомментировать bot.stop_polling() в строке 69
8. Так как возникли проблемы с запуском webhook на локальной машине, а времени на веб-сервер из-за загруженности не хватило, обход ошибок телеги выполняем при помощи try/except, причём с достаточно переменным успехом

Для работы требуются:
PyTorch
TensorFlow
pyTelegramBotAPI 3.6.7
subprocess
threaded 4.0.8.
numpy
scipy
PIL
